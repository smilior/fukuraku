# 投稿パフォーマンス分析

## 6.1 KPIと目標値

**初期（0-1,000フォロワー）の目標：**

| 指標 | 目標値 | 備考 |
|------|--------|------|
| フォロワー増加 | 月+100人 | |
| インプレッション/投稿 | 500以上 | |
| エンゲージメント率 | 3%以上 | (いいね+RT+リプ)/インプ |
| プロフィールクリック | 投稿あたり10以上 | |
| リンククリック | 投稿あたり5以上 | CTA投稿の場合 |

**中期（1,000-5,000フォロワー）の目標：**

| 指標 | 目標値 | 備考 |
|------|--------|------|
| フォロワー増加 | 月+300人 | |
| インプレッション/投稿 | 2,000以上 | |
| エンゲージメント率 | 2%以上 | |
| アプリ登録への転換 | 月20人以上 | UTMパラメータで計測 |

## 6.2 分析の方法

**X Analytics（無料）で確認する項目：**
- 投稿ごとのインプレッション数
- エンゲージメント率（いいね、RT、リプライ、リンククリック）
- プロフィール訪問数
- フォロワーの増減
- 最も反応の良かった投稿

**分析の頻度：**
- 毎日：投稿のパフォーマンスを簡易チェック
- 毎週：週間レポートを作成（スプレッドシートに記録）
- 毎月：月間トレンドの分析、戦略の見直し

## 6.3 伸びた投稿・伸びなかった投稿の分析

**伸びやすい投稿の特徴：**
- 数字が入っている（「5選」「3ステップ」等）
- 画像やスクリーンショットが添付されている
- 共感を呼ぶ内容（あるある、不安の共有）
- 具体的で実用的な情報
- 投稿時間が朝7-8時 or 夜20-22時

**伸びにくい投稿の特徴：**
- URLだけのリンク共有
- 宣伝色が強い投稿
- 専門用語が多い
- 長すぎるスレッド（5ツイート以上）
- 投稿時間が深夜・早朝

## 6.4 A/Bテストの実施方法

**テスト対象：**
- 同じ内容で異なるキャッチフレーズ（曜日を変えて投稿）
- 画像あり vs 画像なし
- 短文（〜100文字）vs 長文（〜280文字）
- CTA文言の比較（「詳細はこちら」vs「無料で試す」）
- 投稿時間の比較（朝 vs 夜）

**記録方法：**
Notionやスプレッドシートで以下を記録：
- 投稿日時
- 投稿内容（タイプ分類）
- インプレッション
- いいね/RT/リプライ数
- エンゲージメント率
- リンククリック数（UTMパラメータ使用）
