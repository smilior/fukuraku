# ブログ記事テンプレート・note/Zenn戦略

## ブログ記事の基本構成（2,000-3,000字）

```
1. タイトル（32文字以内、キーワード含む）
2. 導入部（200字）
   - 読者の悩みに共感
   - この記事で解決できることを予告
3. 目次
4. 本文（H2 x 3-5セクション）
   - 各H2に2-3のH3
   - リスト・表・画像の活用
   - 具体例を必ず含める
5. まとめ（200字）
   - 要点の再確認
   - CTAへの自然な誘導
6. CTA
   - 副楽の紹介（3行以内）
   - 登録リンク
```

## タイトルの書き方

**効果的なタイトルパターン：**

1. **数字入り：** 「副業で経費にできるもの15選【完全版】」
2. **疑問形：** 「副業で20万円超えたら確定申告は必要？」
3. **How to：** 「副業の確定申告のやり方を3ステップで解説」
4. **比較型：** 「freee vs マネーフォワード vs 副楽 副業サラリーマン向け比較」
5. **年度入り：** 「【2025年版】副業の確定申告 完全ガイド」
6. **ターゲット明示：** 「副業サラリーマン必見！確定申告で損しない方法」
7. **ネガティブ型：** 「副業の確定申告を忘れたらどうなる？ペナルティと対処法」

**タイトル作成チェックリスト：**
- [ ] 主要キーワードを含んでいる
- [ ] 32文字以内（Google検索結果に収まる）
- [ ] 読者のメリットが明確
- [ ] 具体的な数字が入っている（可能であれば）
- [ ] 年度が入っている（2025年版等）

## 本文の構成ルール

**H2/H3の使い方：**
- H2：大きなテーマ（3-5個）
- H3：H2の中のサブトピック（各H2に2-3個）
- H4以降：できるだけ使わない（深すぎる構造は読みにくい）

**読みやすくするコツ：**
- 1段落は3-4行以内
- 箇条書きリストを積極的に使う
- 表（テーブル）で情報を整理する
- 重要なポイントは太字で強調する
- 2-3スクロールごとに画像や図を入れる

**内部リンクの設置：**
- 関連する自社ブログ記事へリンク
- 最終段落で副楽へのリンク
- アンカーテキストにキーワードを含める

```markdown
## 1. 副業の確定申告が必要な条件

### 基本ルール：所得20万円超で確定申告が必要

給与所得者（会社員）で副業をしている場合、
副業の「所得」が年間20万円を超えると確定申告が必要です。

### 所得税法上の根拠

所得税法第121条第1項に基づき、
給与所得以外の所得が20万円を超える場合は確定申告が必要です。

---

## 2. 「収入」と「所得」の違い

### 所得 = 収入 - 経費

ここが最も重要なポイントです。

| 用語 | 意味 | 例 |
|------|------|-----|
| 収入 | 売上の合計 | 50万円 |
| 経費 | 事業に必要な支出 | 35万円 |
| 所得 | 収入 - 経費 | 15万円 |

上の例では、収入は50万円ですが、
経費を差し引いた所得は15万円です。
所得が20万円以下なので、所得税の確定申告は不要です。

### 経費にできるものの例

- パソコン、モニター
- インターネット回線（按分）
- サーバー代、ドメイン代
- 参考書籍
- 交通費
- コワーキングスペース代

**経費をしっかり記録することで、
確定申告が不要になるケースもあります。**

---

## 3. 確定申告の手順（3ステップ）

### Step 1: 必要書類を準備する

- 源泉徴収票（会社から受け取る）
- 副業の収入がわかる書類（請求書、振込記録等）
- 経費の領収書・レシート
- マイナンバーカード（e-Taxの場合）

### Step 2: 確定申告書を作成する

- 国税庁の確定申告書等作成コーナー
- e-Tax
- 確定申告アプリ（副楽等）

### Step 3: 申告書を提出する

- e-Taxで電子申告（おすすめ）
- 税務署に郵送
- 税務署に持参

提出期限：毎年2月16日～3月15日

---

## 4. 20万円以下でも申告が必要なケース

### 住民税の申告

所得税の確定申告は不要でも、住民税の申告は必要です。
これは多くの副業サラリーマンが見落としているポイントです。

### 医療費控除・ふるさと納税の申告

副業所得が20万円以下でも、医療費控除やふるさと納税の
確定申告（還付申告）を行う場合は、
副業所得も合わせて申告する必要があります。

---

- 経費をしっかり記録すれば、申告不要になるケースもある
- 20万円以下でも住民税の申告は必要
```

## noteとZennへの投稿戦略

### note：一般向けコンテンツ

**noteの位置づけ：**
- 副業サラリーマンへの直接的なリーチ
- SEOの被リンク獲得
- 副楽のブランディング
- 自社ブログへの流入導線

**投稿テーマ：**
1. 副業×税金の体験談（ストーリー形式）
2. 確定申告の失敗談と学び
3. 副業サラリーマンの1年間の税金記録
4. 確定申告をAIに任せてみた結果

**投稿頻度：** 月2回

**noteの記事構成テンプレート：**
```
タイトル：体験談風（「副業の確定申告で3万円損した話」等）

導入：個人的な体験やエピソード
本文：体験から学んだこと、具体的なノウハウ
結論：読者へのアドバイス
CTA：「副楽で経費管理を楽に → [URL]」

文字数：2,000-3,000字
画像：2-3枚（スクリーンショット、図解等）
```

**note投稿例タイトル：**
- 「副業の確定申告に30時間かかった話。そして3分になった話。」
- 「副業で月5万円稼いで気づいた『税金の壁』」
- 「freeeが難しすぎて挫折した副業サラリーマンが自分でアプリを作った」
- 「確定申告シーズンにパニックにならないための、たった1つの習慣」

### Zenn：エンジニア向け技術記事

**Zennの位置づけ：**
- エンジニアへのリーチ（副業をしているエンジニアは多い）
- 技術スタックのアピール（Next.js, Supabase, AI SDK）
- 個人開発者コミュニティとのつながり
- 採用・協力者募集にもつながる

**投稿テーマ：**
1. 技術選定の裏話（なぜ Next.js + Supabase を選んだか）
2. AI SDK を使ったレシートOCRの実装方法
3. Supabase RLSでのマルチテナント設計
4. Stripe Billing でサブスクリプション決済を実装する方法
5. Vercel AI SDK でOpenAI Vision APIを使う方法
6. 個人開発のアーキテクチャ設計記録

**投稿頻度：** 月1-2回

**Zenn記事構成テンプレート：**
```
タイトル：技術的な内容を明確に
         「Next.js + AI SDKでレシートOCRアプリを作った」

導入：作ったものの概要、技術的な背景
技術選定：なぜその技術を選んだか、比較検討
実装：コードスニペット付きの実装方法
ハマったポイント：トラブルシューティング
結果：パフォーマンス、精度等の結果
まとめ：学びと今後の展望

文字数：3,000-5,000字
コードブロック：5-10個
```

**Zenn投稿例タイトル：**
- 「副業確定申告アプリをNext.js + AI SDKで個人開発した話」
- 「Supabase RLSでユーザーデータを完全に分離する設計パターン」


- Zenn：技術的深掘り、コードスニペット重視
