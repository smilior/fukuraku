# Google Search Console・コンテンツカレンダー

## Google Search Console 設定

### サイト登録手順

1. Google Search Console にアクセス
2. 「プロパティを追加」をクリック
3. 「URLプレフィックス」を選択（推奨）
4. `https://fukuraku.app` を入力
5. 所有権の確認方法を選択：
   - HTML タグ（推奨）→ `<meta>` タグをlayout.tsxに追加
   - DNS レコード → Vercelのドメイン設定で追加
   - HTML ファイル → publicフォルダにファイルを配置

**Next.js での認証タグ設定：**
```typescript
// app/layout.tsx の metadata に追加
export const metadata: Metadata = {
  verification: {
    google: "認証コードをここに入力",
  },
};
```

### サイトマップ送信

1. Search Console の「サイトマップ」セクションを開く
2. `https://fukuraku.app/sitemap.xml` を入力して送信
3. ステータスが「成功」になることを確認

**送信タイミング：**
- 初回設定時
- 新しいブログ記事を公開した後
- サイト構造を大きく変更した後

### インデックス状況の確認

**確認すべき項目：**

1. **カバレッジレポート**
   - 有効なページ数
   - エラーのあるページ
   - 除外されたページ
   - 警告のあるページ

2. **URL検査ツール**
   - 特定のURLがインデックスされているか確認
   - インデックス登録をリクエスト（新規ページの場合）

3. **検索パフォーマンス**
   - 表示回数（インプレッション）
   - クリック数
   - 平均掲載順位
   - CTR（クリック率）

### 定期チェック項目

**週次チェック：**
- [ ] 新規記事がインデックスされているか
- [ ] エラーが発生していないか
- [ ] 検索パフォーマンスの推移

**月次チェック：**
- [ ] 主要キーワードの掲載順位の推移
- [ ] クリック率の改善（タイトル・ディスクリプションの最適化）
- [ ] 新しいキーワードの発見（「検索クエリ」レポート）
- [ ] モバイルユーザビリティの問題がないか
- [ ] Core Web Vitalsのスコア

## コンテンツ更新スケジュール

既存記事の更新も重要なSEO施策です。

- **年1回（10月）：** 全記事の年度更新（「2025年版」→「2026年版」）
- **法改正時：** 関連する記事を即時更新
- **パフォーマンス分析後：** 掲載順位が下がった記事をリライト
- **四半期ごと：** 上位表示されている記事のリンク切れチェック
