# ベータテスター募集・オンボーディング

## 募集の目的

ベータテストの主な目的：
- 実際のユーザーによるフィードバック収集
- バグの早期発見と修正
- 初期ユーザーの確保（将来の有料転換候補）
- 社会的証明（レビュー、口コミ）の獲得
- プロダクトマーケットフィットの検証

## フィードバック収集フォームの設計

Google Forms を使ったフィードバックフォームの設計：

**フォーム項目：**

1. **総合満足度**（1-10点のスケール）
   - 1: 全く満足していない 〜 10: 非常に満足

2. **使いやすさ**（1-5点）
   - 1: とても使いにくい 〜 5: とても使いやすい

3. **レシート読取精度**（1-5点）
   - 1: ほとんど読み取れない 〜 5: ほぼ完璧

4. **最も気に入った機能**（自由記述）

5. **改善してほしい点**（自由記述）

6. **欲しい機能**（複数選択 + 自由記述）
   - [ ] 銀行口座連携
   - [ ] クレジットカード明細取込
   - [ ] 確定申告書の直接作成
   - [ ] 複数年度のデータ管理
   - [ ] 税理士への相談機能
   - [ ] その他（自由記述）

7. **このアプリを友人に勧めますか？**（NPS: 0-10点）
   - 0: 全く勧めない 〜 10: 強く勧める

8. **有料プラン（月額980円）への支払い意欲**（1-5点）
   - 1: 絶対に払わない 〜 5: 喜んで払う

9. **その他のコメント**（自由記述）

## ベータユーザーへの特典設定

**特典の設計方針：**
- 初期20名に対して高い価値の特典を提供
- 副楽の将来的なコスト負担を抑えつつ、感謝を示す
- 口コミ・レビューにつながる特典設計

**特典パターン：**

| 特典 | 内容 | 適用条件 |
|------|------|---------|
| 永久無料枠 | プロプランを永久無料 | 先着20名のベータテスター |
| Founding Member バッジ | プロフィールに特別バッジ表示 | ベータテスト参加者全員 |
| 優先サポート | フィードバックへの優先対応 | ベータテスト参加者全員 |
| 機能リクエスト権 | 開発ロードマップへの提案権 | 有益なフィードバックを提供した方 |

**永久無料枠の実装：**
```sql
-- ベータユーザーフラグをusersテーブルに追加
ALTER TABLE users ADD COLUMN is_beta_user BOOLEAN DEFAULT FALSE;
ALTER TABLE users ADD COLUMN beta_joined_at TIMESTAMPTZ;

-- ベータユーザーはプロプランの制限を適用しない
-- サブスクリプションチェック時にis_beta_userを確認
```

## ベータテストのスケジュール

```
Week 1: テスター募集開始（X投稿、DM対応）
Week 2: テスター招待、オンボーディング
Week 3: テスト期間（フィードバック収集開始）
Week 4: フォローアップ、追加フィードバック収集
Week 5: フィードバック分析、改善優先度付け
Week 6: 改善版の実装・デプロイ
Week 7: 改善版のテスト確認
Week 8: 本リリースへ移行
```
