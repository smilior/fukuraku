# リリース後30日間フォローアップ計画・KPI

## Day 1-7：バグ修正最優先期間

**日次タスク：**
- [ ] 全ユーザーフィードバックの確認と返信
- [ ] バグレポートの確認と優先度付け
- [ ] クリティカルなバグの即日修正
- [ ] サーバーの負荷状況の監視
- [ ] GA4のリアルタイムレポート確認

**対応フロー：**
```
バグ報告
  ↓
重大度の判定
  ├── クリティカル（データ損失、セキュリティ） → 即日修正
  ├── 高（機能が使えない） → 24時間以内に修正
  ├── 中（機能が不完全） → 1週間以内に修正
  └── 低（UI/UXの改善） → バックログに追加
```

**Day 1-7 のKPI：**
- 新規登録者数：目標 30人
- アクティブユーザー数：目標 10人/日
- バグ修正数：報告されたバグの80%を修正
- 平均レスポンスタイム：フィードバックへの返信 2時間以内

## Day 8-14：フィードバック分析期間

**タスク：**
- [ ] 全フィードバックの集計・分析
- [ ] NPS（Net Promoter Score）の計算
- [ ] 最も多い要望の特定
- [ ] ユーザーの離脱ポイントの分析（GA4ファネル分析）
- [ ] 改善項目の優先度付け

**フィードバック分析テンプレート：**

| カテゴリ | 件数 | 主な内容 | 優先度 | 対応予定 |
|---------|------|---------|--------|---------|
| バグ | X件 | ... | 高 | Week 3 |
| UI/UX | X件 | ... | 中 | Week 3-4 |
| 新機能要望 | X件 | ... | 低 | バックログ |
| 褒められた点 | X件 | ... | - | 継続 |

**ユーザーインタビュー（オプション）：**
- アクティブユーザー3-5人にオンラインインタビュー
- 使い方の観察（画面共有してもらう）
- 課題と期待を直接聞く
- 30分程度、Amazonギフト券等のお礼を用意

## Day 15-21：最初の改善版リリース

**改善版で対応すべき項目：**
1. クリティカルなバグの残り全て
2. 最も多かったUI/UXの改善
3. 最も要望の多かった小機能の追加
4. パフォーマンスの改善

**リリースノートのテンプレート：**
```
副楽 v1.1 リリースノート

【改善】
- レシート読取精度を94% → 96%に改善
- ダッシュボードの表示速度を50%高速化
- 経費分類の編集UIを改善

【バグ修正】
- 特定の条件でレシート画像がアップロードできない問題を修正
- 月別レポートの金額計算が一部不正確だった問題を修正

【新機能】
- 経費データのCSVエクスポート機能（プロプラン）

皆様のフィードバックに基づいて改善しました。
引き続きご意見をお聞かせください。
```

**X での告知：**
```
副楽 v1.1 をリリースしました。

皆様のフィードバックを反映して：
- レシート読取精度 94% → 96% に改善
- ダッシュボードの表示速度 50% 高速化
- CSV エクスポート機能を追加

今後も改善を続けます。
フィードバックお待ちしています。
```

## Day 22-30：有料転換施策の実施

**有料転換を促す施策：**

1. **フリープランの制限到達通知**
   ```
   今月のレシート登録が5枚に達しました。
   プロプランにアップグレードすると、
   レシート登録が無制限になります。
   月額980円 → アップグレードする
   ```

2. **期間限定キャンペーン**
   ```
   リリース記念キャンペーン
   プロプラン初月50%オフ（月額490円）
   期間：〜20XX年XX月XX日
   ```

3. **メールでのナーチャリング**
   ```
   Day 3: ウェルカムメール（使い方ガイド）
   Day 7: Tips メール（経費にできるもの一覧）
   Day 14: 成功事例メール（ベータユーザーの声）
   Day 21: プロプランの紹介メール
   Day 28: 期間限定キャンペーン告知
   ```

4. **機能制限の活用**
   - フリープランでは月5枚のレシート制限
   - 制限に達した時点でアップグレード誘導
   - 制限直前にメール通知
