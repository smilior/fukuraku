## KPI スプレッドシート更新手順

```markdown
## KPI 記録手順（毎月1日実施）

1. Google Sheets の「副楽 KPI トラッカー」を開く
2. 新しい月の行を追加
3. 以下の値を入力:

| 指標 | 入力元 | 列 |
|-----|-------|---|
| MRR | Stripe Dashboard > MRR | B |
| アクティブ会員数 | Stripe > Subscriptions | C |
| チャーン率 | Stripe > Cancellations / 前月アクティブ | D |
| ARPU | MRR / アクティブ会員数 | E |
| MAU | GA4 > ユーザー概要 | F |
| 新規登録数 | Supabase > auth.users count | G |
| 転換率 | 新規有料 / 新規登録 | H |
| OCR利用回数 | GA4 > receipt_uploaded イベント数 | I |
| オンボーディング完了率 | GA4 > ファネル | J |

4. グラフが自動更新されることを確認
5. 目標値との乖離が大きい項目にコメントを追加
```

## 月次レビューミーティングアジェンダ

```markdown
## 月次レビューアジェンダ（30分）

1. **KPI サマリー**（5分）
   - MRR、チャーン率、MAU の前月比
   - 目標達成率

2. **ユーザー獲得分析**（5分）
   - 新規登録の流入チャネル
   - 効果的だった施策

3. **プロダクト利用状況**（10分）
   - 主要機能の利用状況
   - ユーザーの行動パターン変化
   - ファネル改善ポイント

4. **チャーン分析**（5分）
   - 解約ユーザーの特徴
   - リテンション施策の効果

5. **翌月のアクションアイテム**（5分）
   - 改善施策の優先順位
   - 確定申告シーズン準備（10-12月）
```

## 実装の優先順位
