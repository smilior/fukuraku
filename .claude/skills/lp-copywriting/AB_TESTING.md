# A/Bテスト戦略

## テスト優先度

**高優先度（売上への直接影響が大きい）：**
1. Heroのキャッチコピー
2. CTAボタンの文言
3. 料金プランの見せ方

**中優先度：**
4. 問題提起セクションの表現
5. 社会的証明の表示方法
6. FAQの内容と順序

**低優先度：**
7. 色やフォントサイズ
8. 画像の選択
9. セクションの順序

---

## キャッチコピーのA/Bテスト

**テスト案1：**
- A案：「レシートを撮るだけ。確定申告の準備が終わる。」
- B案：「確定申告の準備、30時間→3分。」
- 計測指標：CTA クリック率、登録転換率

**テスト案2：**
- A案：「副業の確定申告、もうAIに任せませんか？」
- B案：「副業サラリーマン専用。確定申告AIアプリ。」
- 計測指標：CTA クリック率、直帰率

---

## CTAボタンのA/Bテスト

**テスト案：**
- A案：「無料で始める」（緑色ボタン）
- B案：「今すぐ無料で試す」（青色ボタン）
- C案：「30秒で登録完了」（オレンジ色ボタン）
- 計測指標：クリック率、登録完了率

---

## 料金の見せ方のA/Bテスト

**テスト案：**
- A案：月額表示「980円/月」
- B案：日額表示「1日あたり約32円」
- C案：年額表示「年間11,760円（月額980円）」
- 計測指標：有料プランへの転換率

// Vercel の Edge Config を使った簡易A/Bテスト

## テスト結果の評価

**統計的有意性の確認：**
- 最低サンプルサイズ：各案 100クリック以上
- テスト期間：最低2週間（曜日変動を含む）
- 有意水準：p < 0.05

**評価のポイント：**
- CTA クリック率だけでなく、最終的な登録転換率まで追跡
- モバイルとデスクトップで別々に分析
- 新規訪問者とリピーターで別々に分析
