## はじめに

副業の確定申告に30時間かかった。

これが、副楽を作り始めたきっかけです。

本業はエンジニアとして会社に勤めながら、副業でも収入を得ている、いわゆる「副業サラリーマン」。副業自体は楽しかったのですが、年に一度の確定申告だけは本当に苦痛でした。

この記事では、「自分の課題を自分で解決するアプリを作った」ストーリーを、開発の裏側も含めてお話しします。

## 確定申告で地獄を見た話

去年の2月のこと。確定申告の期限が迫る中、私はExcelと格闘していました。

**やっていたこと：**

1. 財布やカバンの中からレシートを集める（すでに一部紛失）
2. レシートの内容をExcelに1枚ずつ手入力
3. 「この支出は経費になるの？」と国税庁のサイトで調べる
4. 経費カテゴリの分類が正しいか何度も確認
5. 合計額が合わなくて再計算

これを仕事終わりの夜や休日にちまちまやって、トータル30時間。

一番つらかったのは「経費の分類」です。通信費？消耗品費？雑費？正解がわからないから、いちいちネットで調べる。同じような質問をしているYahoo!知恵袋の回答も、人によって言っていることが違う。

freeeやマネーフォワードも試しました。でも、フリーランスや法人向けの機能が多すぎて、副業サラリーマンの私にはオーバースペック。設定項目が多くて、初期設定の途中で挫折しました。

「副業サラリーマン専用の、もっとシンプルなアプリがあればいいのに。」

## 「ないなら作ろう」の始まり

エンジニアの職業病というか、「ないなら自分で作ればいい」と思ってしまうのが性分です。

最初に考えた要件はシンプルでした。

1. レシートを撮影するだけで経費が記録できる
2. AIが経費のカテゴリを自動で判定してくれる
3. 副業の所得が20万円を超えそうかリアルタイムでわかる
4. 確定申告に必要なデータをCSVで出力できる

たったこれだけ。仕訳帳も、貸借対照表も、請求書発行も不要。副業サラリーマンに必要な機能だけに絞りました。

## 技術選定の話

個人開発者として大切にしたのは「できるだけ管理コストの低いスタック」を選ぶこと。

**選んだ技術スタック：**

- **フロントエンド:** Next.js（App Router）
- **UI:** shadcn/ui + Tailwind CSS
- **バックエンド・認証・DB:** Supabase
- **AI（レシートOCR）:** Vercel AI SDK + OpenAI Vision API
- **決済:** Stripe
- **ホスティング:** Vercel

全部「開発者フレンドリー」なサービスで揃えました。

特にSupabaseは、認証、データベース、ストレージがまとめて使えるので、個人開発には最高です。Firebase的な手軽さと、PostgreSQLの安心感の両方がある。

AIのレシート読み取りには、OpenAIのVision API（GPT-4o）を使いました。驚いたのはその精度。レシートの写真を送るだけで、日付、金額、品目、店名をほぼ正確に読み取ってくれる。しかも経費カテゴリの判定まで。

画像の前処理（傾き補正とか）が必要かなと思っていたのですが、全然不要でした。AIに丸投げするだけで94%の精度。技術的な工夫より、プロンプトの調整の方が効果がありました。

## 開発中に学んだこと

### 学び1: 最小限でリリースする

最初は「あれもこれも」と機能を追加したくなりました。グラフ表示、PDF出力、多通貨対応...。でも、全部「後でやる」リストに入れました。

まずは「レシートを撮って記録する」というコア機能だけでリリース。ユーザーの反応を見てから、追加機能を決める。

これは正解でした。実際にベータテスターに使ってもらうと、自分が想定していなかった改善点が山ほど出てきたので。

### 学び2: 個人開発のコストは思ったより安い

月額の運用コストは約7,500円（$51）。

- Vercel Pro: $20
- Supabase Pro: $25
- OpenAI API: ~$5
- ドメイン: ~$1

月額980円のプランなら、8人の有料ユーザーで黒字化。個人開発の魅力は、この損益分岐点の低さにあります。

### 学び3: 自分が最初のユーザー

「このアプリに需要はあるのか？」と不安になることもありました。でも、少なくとも1人のユーザー（自分）は確実にいる。

自分の課題を解決するアプリを作っているから、機能の優先度判断も迷わない。「自分が使うか？」という基準で判断できるのは、個人開発ならではの強みです。
