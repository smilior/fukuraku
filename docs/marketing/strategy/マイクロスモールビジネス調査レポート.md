## 核心メッセージ

> **月7万円 → 月20万円への道は、アプリを増やすことではない。ビジネスモデルを変えることだ。**

必要なのは以下の3つの転換：
1. **収益モデルの転換** → 低単価×量　から　高単価×継続課金へ
2. **マーケティングチャネルの確立** → 作るスキルから届けるスキルへ
3. **選択と集中** → 分散をやめ、1つの柱を太くする

## 第5章：やってはいけないこと（警告）

| ❌ 禁止事項 | 理由 |
|---|---|
| 新しいAIラッパーアプリを作る | スイッチングコストゼロ、3ヶ月で消える |
| アプリを増やし続ける | 「生産的な逃避」。全部中途半端になる |
| 低単価×量で勝負する | 月10万円の天井を超えられない |
| 汎用AI系ニッチに参入 | Google・OpenAIと直接競合 |
| 複数プロダクトを同時並行開発 | 月7万円の段階では選択と集中が最優先 |
| 「不労所得」幻想を持つ | 月10万円以上は能動的な運営が必要 |
| APIコストを無視する | 売上が増えるほど赤字になる場合がある |

## 第6章：勝てる条件3つ

### 条件①：差別化要素 = 「業界知識 × AI × 実体験」の掛け合わせ
- 技術だけでは差別化にならない（バイブコーディングは誰でもできる時代）
- **自分だけの実体験・専門知識**がなければ3ヶ月でコピーされる
- 「月7万円稼いだ」という実績はコピー不可能

### 条件②：参入障壁 = データ・ワークフロー・スイッチングコスト
- コードはもうモートにならない（AIがコア機能を複製できる時代）
- 有効な参入障壁：ユーザーデータの蓄積・業務ワークフローへの組み込み・コミュニティ

### 条件③：継続収益構造 = サブスク + 低チャーン
- 一回売り切りモデルだけでは月10〜20万円の安定は困難
- 継続的に価値を提供する仕組みが必要
- 継続利用の理由が弱いサブスクは解約率が高く赤字になる
